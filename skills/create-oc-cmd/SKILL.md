# create-oc-cmd

调用此 skill 时，在**当前用户主目录**（`$env:USERPROFILE`，即 `C:\Users\<User>`）下创建一个 PowerShell 脚本，脚本内容为：切换到**调用此 skill 时的工作目录**，然后启动 `opencode`。

## 执行步骤

1. 用 bash 工具确定当前工作目录与用户主目录：
   ```powershell
   $cwd = (Get-Location).Path
   $home = $env:USERPROFILE
   ```

2. 基于当前工作目录的目录名生成脚本文件名（小写、空格转连字符），默认放在主目录：
   - 文件名：`oc-<dirname>.ps1`（例如当前目录 `trace-analysis` → `oc-trace-analysis.ps1`）
   - 完整路径：`$env:USERPROFILE\oc-<dirname>.ps1`
   - 若用户指定了其它名称或路径，按用户指定。

3. 用 Write 工具在该路径写入脚本，内容如下（将 `<CWD>` 替换为第 1 步得到的实际路径，原样写入、不要插值）：
   ```powershell
   # Launcher generated by create-oc-cmd skill
   # Switches to the project directory and starts opencode.
   Set-Location -LiteralPath '<CWD>'
   opencode
   ```
   - 路径用单引号包裹，避免变量插值。
   - 若路径本身含单引号，则改用双引号并把内部单引号翻倍转义。

4. 完成后向用户报告：
   - 脚本的完整路径
   - 运行方式：在 PowerShell 中执行 `& "<脚本路径>"` 或直接双击；若主目录在 PATH 中也可直接输入脚本名。
   - 提醒：以后运行该脚本即可回到此项目目录并启动 opencode。

## 注意

- 不要修改用户的 PATH 或注册表，只创建一个 .ps1 文件。
- 脚本记录的是**调用此 skill 时**的工作目录，项目迁移后需重新生成。
- 若 `opencode` 未在 PATH 中，脚本会报错；请先确认 `opencode` 可全局调用。
